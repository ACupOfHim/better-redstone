# Better Redstone！— Minecraft 红石工程模组设计文档

## 一、概述

本模组旨在为 Minecraft 红石系统提供一套完整的工程设计解决方案，包含新型导线、逻辑封装、可视化设计环境、自动化制造与部署系统。玩家可在红石工坊中自由设计立体电路，封装为芯片，在生存模式中铸造并部署使用。

## 二、核心导线系统

### 强化红石
- **合成**：红石×3 + 铜锭×3 → 强化红石×3
- **物理特性**：独立碰撞箱，不依附任何方块即可悬空存在，可单独向上向下垂直传输
- **模型**：纯色管，四面显示红石信号强度数字 0~15
- **信号损耗**：每格损耗一级（与原版红石粉一致）
- **放置规则**：一个方块空间内只能放置一根

### 超导红石
- **合成**：强化红石 + 石英 → 超导红石×1
- **物理特性**：与强化红石完全一致，独立碰撞箱，可垂直传输
- **信号损耗**：无信号损耗，任意距离保持原始强度

### 颜色系统
- 玩家可为导线自由选择颜色（通过调谐仪或手持染料右键）
- **连接规则**：同色 **且** 同种类才可连接；不同颜色或不同种类的线即使物理接触也不导通
- **单根连续性**：单根连续的线从头到尾必须是同一种颜色 + 同一种材质
- **跨种类/跨颜色转换**：不允许直接连接，需封装单输入单输出的状态跟随器芯片作为转换接头

## 三、强化红石元件

- **合成规则**：所有原版红石元件配方中的红石粉替换为强化红石
- **中继器**：独立方块实体，可被调谐仪调整方向，新增零延迟档位
- **比较器**：独立方块实体，可被调谐仪调整方向
- **红石火把、按钮、拉杆、压力板等**：配方升级，性质继承

## 四、调谐仪

### 功能
1. 右键依次点击导线或元件，将其标记为已选中
2. 两条导线之间设置为连通或断开
3. 调整导线或红石器件的朝向（热更改，无需敲掉重放）
4. 修改选中导线的颜色
5. 铸造完成后必须用调谐仪预览并放置芯片（防 Bug 设计）

### 交互
- **取消选中**：默认按键 Alt+C（集成到 MC 按键绑定系统，玩家可自定义）
- **放置方式**：类似投影模组，虚拟模型悬浮，调整位置与朝向，热键确认放置

## 五、红石工坊（设计环境）

- **入口**：主菜单第三选项（与"单人游戏""多人游戏"并列），命名为"红石工坊"
- **进入方式**：像原版选择世界一样，进入原理图列表
- **游戏模式**：创造模式，但物品栏受限
- **可用方块**：所有红石元件（含强化版）+ 不同颜色的实体与非实体标记方块 + 注释方块
- **注释方块**：右键可阅读和编辑注释文字，用于记录设计思路和引脚映射
- **建造方式**：完全继承原版三维建造，玩家自由移动放置
- **核心优势**：多层电路不会串线，可搭建榫卯结构的立体电路

### I/O 方块
- 专用输入方块和输出方块，充当芯片与外界交互的引脚
- **引脚规则**：每个 1×1 的面只能放一个引脚，芯片最小尺寸受引脚数量限制
- **引脚映射**：保存前，玩家设置哪个面的输出方块对应哪个信号输出
- **节点引出**：需要某个内部节点对外输出时，从该节点拉线连接到输出方块即可

## 六、芯片封装与文件

- **保存**：在红石工坊中保存为独立的原理图文件
- **文件格式**：NBT（与原版原理图一致）
- **文件扩展名**：`.rsc` 或 `.chip`（待定）
- **存放位置**：`.minecraft/schematics` 或模组专属文件夹
- **文件内容**：元数据（名称、作者、描述）+ 空间尺寸 + 方块数据 + 引脚定义 + 注释数据
- **官方预设**：提供基础电路库（与门、或门、交叉路由、状态跟随器等），供玩家直接使用

### 真值表穷举（核心性能优化）
- 封装时，系统自动穷举所有输入组合，生成真值表
- **时序逻辑支持**：穷举状态为"输入 + 当前状态 → 输出 + 下一状态"，状态机穷举
- **运行时**：直接查表输出，无需模拟内部物理信号传播，性能开销接近于零
- **输入限制**：无硬性上限，引脚超过 16 个时弹出提醒
- **穷举过程**：在芯片铸造时完成，消耗客户端/单机性能，不影响运行时服务器性能

## 七、芯片铸造台（制造系统）

### 合成表
```
钻石  下界合金碎片  钻石
木板  红石块       木板
平滑石 平滑石       平滑石
```
产出：一个芯片铸造台

### 环境要求
上表面必须裸露在空气中（不能被任何方块遮挡），其他五个面可连接木盒

### UI 与队列
- **原理图选择**：显示玩家封装和官方预设两大类
- **成本显示**：选中后显示制造成本清单（原理图中所有材料，1:1 复刻）
- **引脚成本**：输入和输出引脚固定计为两个金锭
- **递归拆解**：若原理图中包含子芯片，玩家可选择拆解为原材料（一键展开）
- **时间累加**：拆解子芯片时，子芯片的制造时间也计入总时间
- **队列数量**：最多五个独立队列
- **输出栏**：五个可堆叠格子，对应五个队列
- **制造数量**：每个队列可任意设置制造数量
- **材料消耗**：制造过程中随时间等比例消耗材料（匀速消耗）
- **暂停机制**：材料不足时自动暂停，补充后自动恢复
- **原料总览**：UI 左上角/右上角显示所有已连接木盒中的原料总数

### 时间计算
- **基准**：每个 1×1×1 方块体积 = 1 分钟制造时间
- **总时间**：主芯片体积 × 1 分钟 + 所有被拆解的子芯片自身制造时间之和
- **UI 提示**：选择拆解时，显示拆解链路上每一层的时间累加
- **穷举时间**：封装穷举真值表的时间单独计算，在铸造过程中完成，UI 中显示进度

### 能量系统
- **输入燃料**：仅接受木炭、煤炭、煤炭块、岩浆桶、烈焰棒
- **能量比例**：与原版熔炉一致
- **消耗速率**：为原版熔炉的两倍（燃烧时间翻倍消耗）
- **共享方式**：全局共享，五个队列共用一个能量池

## 八、木盒（存储与供料系统）

### 合成表
```
铁块 金块 铁块
金块 木桶 金块
铁块 金块 铁块
```
产出：一个木盒

### 特性
- **可合并**：类似机械动力的集装箱，多个木盒可拼接成大木盒
- **存储容量**：一格木盒 = 一个大箱子（54 格），线性叠加
- **自动拆包**：支持自动拆解潜影盒，内部物品弹出到存储空间
- **连接方式**：物理相邻即可连接铸造台或其他木盒
- **作用**：为铸造台提供扩充的材料存储空间和自动供料

## 九、生存模式部署

- **放置方式**：必须使用调谐仪预览放置，不可手动右键放置
- **放置交互**：调谐仪进入预览模式，虚拟模型显示，调整位置与方向，热键确认
- **工作原理**：输入引脚接收红石信号 → 查表运算 → 输出引脚发射信号
- **兼容性**：与原版红石元件完全兼容，可连接普通红石粉、中继器、活塞等

## 十、性能优化规则

| 元件 | 引发亮度更新 | 视觉反馈 |
|------|-------------|---------|
| 红石灯 | ✅ 是 | 真正发光 |
| 铜灯（原版） | ❌ 否 | 纹理中部红点 + 亮态/暗态纹理切换（假发光） |
| 红石火把 | ❌ 否 | 纹理颜色变化（亮态/暗态，假发光） |
| 中继器 | ❌ 否 | 信号强度数字显示 |
| 比较器 | ❌ 否 | 信号强度数字显示 |
| 红石块 | ❌ 否 | 无变化 |
| 强化红石导线 | ❌ 否 | 四面显示信号强度数字 |
| 其他红石元件 | ❌ 否 | 对应状态显示 |

**核心原则**：红石灯是整个体系中唯一真正触发光照引擎更新的红石元件。其他所有元件（包括铜灯和红石火把）只做纹理或颜色层面的状态切换，不触发光照引擎重算。芯片封装后穷举真值表，运行时查表输出，从根本上解决大型电路的卡顿问题。

## 十一、完整闭环流程

1. **红石工坊（设计阶段）**：自由搭建立体电路 → 放置 I/O 方块 → 设置引脚映射 → 添加注释 → 保存为 `.rsc` 原理图文件
2. **芯片铸造台（制造阶段）**：选择原理图（自制/官方预设）→ 查看成本和时间 → 准备材料（木盒供料 + 能量输入）→ 队列设置（最多 5 队列，可设数量）→ 穷举真值表 → 封装为逻辑黑箱 → 产出芯片物品
3. **调谐仪（部署阶段）**：预览虚拟模型 → 调整位置与朝向 → 热键确认放置
4. **生存模式（使用阶段）**：输入引脚接收红石信号 → 查表瞬时输出 → 驱动活塞、发射器、红石灯等原版/模组设备

## 十二、设计哲学

- **不取代原版**：芯片只输出红石信号，不提供动力或不破坏方块，保留原版机械建造乐趣
- **直观胜过复杂**：纯色管 + 四面数字显示信号强度，颜色即规则（同色同种才可连接）
- **性能优先**：真值表穷举 + 禁止无关亮度更新，大型电路零卡顿
- **自由优先原则**：系统不设硬性上限，只在复杂度超过阈值时提醒玩家，由玩家自主决定
- **防呆设计**：调谐仪强制预览放置、组合键取消选中、材料不足自动暂停
- **成本平衡**：材料成本 + 能量消耗 + 时间成本，生存模式数值自洽
- **保留 MC 灵魂**：三维建造手感、红石逻辑核心、与原版元件完全兼容
- **工程化思维**：从设计到制造到部署的完整闭环，递归拆解支持大规模复用

## 十三、技术落地建议

- **游戏版本**：Minecraft 1.21.1（稳定且生态成熟）
- **模组加载器**：Fabric（轻量、高性能、对复杂逻辑友好）
- **开发工具**：JDK 17+ + IntelliJ IDEA 社区版
- **美术资源**：需使用专门的 AI 像素画生成工具或社区合作

## 十四、关于穷举真值表的重要补充

- 穷举过程发生在芯片**铸造阶段**，而非封装保存阶段
- 封装保存时只保存原理图结构，不执行穷举
- 穷举在铸造时执行，消耗的是玩家进行铸造操作时本地电脑的性能
- 穷举完成后，真值表和状态转移表被写入芯片物品的 NBT 数据中
- 芯片物品是自包含的，可随意携带、交易、存放
- 运行时只查表，完全不涉及原版红石信号传播模拟
- 穷举时间与引脚数和内部状态数正相关，系统会提前估算并提示玩家
- 玩家可取消正在进行的穷举，但已消耗的时间不退回
